## 1. 网络边缘 (The Network Edge)
* **定义**：我们每天互动的设备，被称为**端系统 (End Systems)** 或 **主机 (Hosts)**。
* **设备类型**：
    * **客户端**：个人 PC、智能手机、笔记本电脑、平板。
    * **服务器**：数据中心的大型计算机。
    * **物联网 (IoT)**：智能家电、监控摄像头等。
* **交互模式**：
    * **C/S 模式 (Client-Server)**：客户端向服务器请求服务。
    * **P2P 模式 (Peer-to-Peer)**：设备间直接通信，如 BitTorrent。

## 2. 接入网 (Access Networks)
为了将网络边缘的设备接入核心网络（互联网），我们有多种接入方式：

### 家庭接入 (Residential Access)
* **DSL (数字用户线)**：
    * 利用现有的**电话线** (双绞线) 传输。
    * **特点**：专用线路，不与其他用户共享带宽。
* **电缆网络 (Cable Internet)**：
    * 利用**有线电视**基础设施 (HFC - 混合光纤同轴电缆)。
    * **特点**：共享广播介质，用户共享带宽。
* **FTTH (光纤到户)**：
    * 光纤直接铺设到家庭。
    * **特点**：提供极高的带宽和速度。
* **5G 固定无线 (5G Fixed Wireless)**：
    * 通过 5G 信号向特定位置（如家庭）提供宽带服务，无需铺设线缆。

### 企业与家庭局域网 (Enterprise & Home Access)
* **以太网 (Ethernet)**：
    * 最流行的有线局域网技术。
    * 使用双绞线连接到以太网交换机。
    * 特点：稳定、高速 (100Mbps, 1Gbps, 10Gbps)。
* **Wi-Fi (无线局域网)**：
    * 基于 IEEE 802.11 标准。
    * 通过无线接入点 (AP) 连接移动设备。

### 广域无线接入 (Wide-Area Wireless Access)
* **3G / 4G (LTE) / 5G**：
    * 由电信运营商部署的蜂窝网络。
    * **特点**：覆盖范围广（数公里），支持移动性。
    * **5G**：相比前代，提供更高带宽、更低延迟和更大连接密度。

## 3. 网络核心 (The Network Core)
网络核心由互联的路由器网络组成，主要任务是数据传输。

### 分组交换 (Packet Switching)
* **机制**：将长报文划分为较小的数据块，称为**分组 (Packets)**。
* **存储转发 (Store-and-forward)**：路由器必须接收到整个分组后，才能开始向下一个链路传输。
* **排队时延与丢包**：
    * 如果到达速率 > 链路传输速率，分组会在缓冲区排队（排队时延）。
    * 如果缓冲区满了，新到达的分组会被丢弃（丢包）。
* **转发表与路由协议**：
    * **转发表**：路由器根据分组的目的IP地址，查询转发表决定输出链路。
    * **路由选择协议**：自动构建和更新转发表的算法。

### 电路交换 (Circuit Switching)
* **机制**：在通信双方建立一条专用的、端到端的连接（如传统电话网）。
* **特点**：
    * 资源（带宽）预留，保证性能。
    * **复用技术**：频分复用 (FDM) 和 时分复用 (TDM)。
* **对比**：
    * 分组交换效率更高（统计多路复用），适合突发数据。
    * 电路交换适合实时、恒定速率的业务，但资源利用率低。

## 4. 分组交换网中的时延、丢包和吞吐量
* **节点总时延 ($d_{nodal}$)** = $d_{proc} + d_{queue} + d_{trans} + d_{prop}$
    1.  **处理时延 ($d_{proc}$)**：检查首部错误、决定输出链路（微秒级）。
    2.  **排队时延 ($d_{queue}$)**：在缓冲区等待传输的时间（取决于拥塞程度）。
    3.  **传输时延 ($d_{trans}$)**：将分组推向链路所需时间 = $L/R$ (分组长度/链路带宽)。
    4.  **传播时延 ($d_{prop}$)**：信号在介质中传播的时间 = $d/s$ (距离/传播速度)。

## 5. 协议层次 (Protocol Layers)
互联网采用 **TCP/IP 五层模型**：
1.  **应用层 (Application)**: HTTP, SMTP, FTP, DNS (报文 Message)
2.  **运输层 (Transport)**: TCP, UDP (报文段 Segment)
3.  **网络层 (Network)**: IP, 路由协议 (数据格 Datagram)
4.  **链路层 (Link)**: Ethernet, WiFi, PPP (帧 Frame)
5.  **物理层 (Physical)**: 传输比特流 (Bit)

## 6. 网络安全 (Network Security)
* **恶意软件 (Malware)**：病毒 (Virus)、蠕虫 (Worm)。
* **拒绝服务攻击 (DoS/DDoS)**：通过大量流量淹没目标，使其无法提供服务。
* **嗅探 (Packet Sniffing)**：被动截获流经网络的数据（如 Wireshark）。
* **IP 哄骗 (IP Spoofing)**：伪造源 IP 地址，冒充他人发送数据。
